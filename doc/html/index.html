What is Perl webget project?

First, Perl is a anguage for getting your job done. Developed by Larry Wall, who must be by now the most famous programming language for the web age. Almost every little utility can be best and easiest written using odd-looking Perl language. Perl is also a cross platform language, meaning that you can run the programs under Unix, Linux and Windows environments. Learn more about Perl at http://www.perl.org and Windows Perl downloads are available at http://www.activestate.com

Perl program mywebget.pl is a tool that can help you to keep track of programs and package releases around the net. The major differences to popular wget(1) utility are as follows:

  • Configuration file(s). Program supports file downloads as with wget(1) by specifying the URLs in the command line. But the advantage is to automate the periodic downloads from your favourite locations by adding entries to configuration file(s). You refer to named tag in order to download new files from developer's site.

    The configuration file can use unlimited number of hierarchy levels: like a) "linux" b) sub-level "linux-gnome" c) more lower level xxx. If you request tag "linux", everything under it are downloaded.

  • Heuristics. As developers release new packages, the package name is subject to changes. The de facto packaging standard is package-version.suffix. An example: famous editor Emacs is distributed under name emacs-21.1.tar.gz. You do not need to know the exact URL, because program is able to find the latest version from FTP directories or HTTP pages. If you had instructed to download emacs-19.28.tar.gz and supplied option option --new, the newest kit would have been found. Record the initial URL to a configuration file, instruct to monitor newer versions, and you can start keeping your favourite programs up to date

  • Content check. If a page displys archive of many files (like directory listing), it is possible to download only files whose content match a regular expression. For example, it is possible to march author's name in the file with option --Regexp-content='(?i)Author:\s*Mr.\s*Foo'

  • Extract. After download, program can unpack the downloaded package in a civilized manner. If archive includes a root directory, then everything is ok, if not, root directory is created upon unpack. This is desireable, because some archives may not include proper package-version/ root directory.

  • Install. Last, it would be nice if program invoked make(1) or some other utility to build and install everything. This feature is not yet implemented, but is is on the sketchboard. Keep on eye on this project.

All in all, mywebget.pl is designed to be your companion to keep in touch with the changing package releases in this software world. The aim is to develop utility which can go to an URL, sniff around a bit to make the decision what to download, fetch and finally install everything into the system. Debian Linux has developed .deb archives and Red Hat Linux can use .rpm respectively, but there is a need to handle .tar.gz, .tgz, .tar.bz2 and othe rindividual file downloads as well. Give mywebget.pl a try and it may make your URL tracking job easier.

Test run - creating Emacs site-lisp

First, Emacs is The Editor, The IDE, that helps you to get your work done. Refer to http://tiny-tools.sourceforge.net => Emacs resource page for more information on Emacs downloads, documentation, add-on package descriptions etc.

This project's includes a configuration files to generate Emacs /usr/share/site-lisp/net package hierachy (the install location is configurable). The configuration file mywebget-emacs.conf records every knwon Emacs Lisp developer around the world and contains instructions how to download all their Emacs packages and how to install them . Follow the inctruction in the configuration file, how to set it up in your system.

What happens after running the Emacs configuration file? Peek mywebget-emacs-directory-list.txt for complete listing what hierarchies are generated and how would the generated directory structure look like. The download count of files is roughly 8100 including 140M lisp code which will be download and unpacked.

Here is the list of known packages and Emacs Lisp developers (2002-08-02) who publish their code somewhere in the Net. This list does not include projects which are already available via CVS or other public version control means. If you are developing Emacs Lisp and you do not appear in this list, please join in and send email to the maintainer with following information: a) your Email address. Send address that is the most permanent and reliable, so that you can be contacted to keep the configuration in correct form b) URL(s) to your work.

Below, you also see how the hierarchies are defined. Requesting tag level 1, elisp, you get everything on Emacs Lisp. First dry-run the above command and if you're satisfied with the paths add option --Create-paths 

    bash$ perl mywebget.pl --verbose --Tag elisp
